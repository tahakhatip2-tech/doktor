# Hakeem Jordan - نظام إدارة العيادات الذكي 🏥

نظام متطور وشامل لإدارة العيادات الطبية، مدعوم بتقنيات الذكاء الاصطناعي ليعمل كسكرتير آلي ذكي عبر واتساب. يهدف النظام لأتمتة حجوزات المواعيد، إدارة ملفات المرضى، وتوفير تجربة مستخدم سلسة وعصرية.

## 🌟 مميزات النظام
1.  **سكرتير آلي ذكي (WhatsApp AI):** يربط واتساب العيادة بذكاء اصطناعي للرد على استفسارات المرضى وحجز المواعيد تلقائياً.
2.  **إدارة المواعيد:** تقويم تفاعلي، حجز، تعديل، وإلغاء المواعيد بسهولة.
3.  **إدارة المرضى:** سجلات طبية شاملة، وصفات إلكترونية، وتاريخ مرضي.
4.  **لوحة تحكم (Dashboard):** إحصائيات دقيقة، رسوم بيانية، وتنبيهات فورية.
5.  **تقارير مالية:** تتبع الدخل والمدفوعات.
6.  **تصميم عصري:** واجهة مستخدم (UI/UX) احترافية تدعم الوضع الليلي واللغة العربية بالكامل.

## 🛠️ بيئة العمل والتقنيات (Tech Stack)

### الواجهة الأمامية (Frontend)
-   **React + Vite:** لأداء فائق السرعة وتجربة استخدام سلسة.
-   **Tailwind CSS:** لتصميم واجهات عصرية ومتجاوبة.
-   **Shadcn/UI:** مكتبة مكونات احترافية.
-   **TanStack Query:** لإدارة حالة البيانات والاتصال بالسيرفر.

### الخادم (Backend)
-   **NestJS:** إطار عمل قوي ومنظم وقابل للتوسع.
-   **Prisma ORM:** للتعامل مع قاعدة البيانات بكفاءة وأمان.
-   **Baileys (WhatsApp Web API):** لإدارة جلسات واتساب برمجياً.
-   **Google Gemini AI:** عقل السكرتير الآلي لتحليل النصوص والرد الذكي.

### قاعدة البيانات (Database)
-   **Supabase (PostgreSQL):** قاعدة بيانات سحابية قوية وموثوقة.

### الربط والنشر (DevOps)
-   **Ngrok:** لإنشاء نفق آمن (Secure Tunnel) يربط السيرفر المحلي بالعالم الخارجي (واتساب وفيرسال).
-   **Vercel:** لاستضافة الواجهة الأمامية.

## 🚀 أوامر التشغيل (How to Run)

لتشغيل المشروع كاملاً على جهازك، اتبع الخطوات التالية بالترتيب:

### 1. تشغيل السيرفر (Backend)
هذا هو "العقل" المحرك للنظام.
```bash
# الانتقال لمجلد السيرفر
cd server-nestjs

# تثبيت الملحقات (مرة واحدة فقط)
npm install

# تشغيل السيرفر في وضع التطوير
npm run start:dev
```
*سيعمل السيرفر على الرابط: `http://localhost:3000`*

### 2. تشغيل الواجهة (Frontend - Local)
لتشغيل واجهة المستخدم على جهازك.
```bash
# ارجع للمجلد الرئيسي (إذا كنت داخل server-nestjs)
cd ..

# تشغيل الواجهة
npm run dev
```
*ستعمل الواجهة على الرابط: `http://localhost:8080`*

### 3. ربط السيرفر بالإنترنت (Ngrok) 🌍
هذه الخطوة **ضرورية جداً** لكي يعمل:
*   تطبيق واتساب (يستقبل الـ Webhooks).
*   الموقع المرفوع على فيرسال (يتصل بجهازك).

```bash
# في نافذة جديدة
ngrok http 3000
```
سيظهر لك رابط يشبه: `https://tsunamic-unshameable-
maricruz.ngrok-free.dev`. **هذا هو الرابط السحري!**

## 🔗 طريقة تحديث الرابط (عند تشغيل Ngrok من جديد)
بما أنك تستخدم النسخة المجانية من Ngrok، سيتغير الرابط في كل مرة تغلق فيها البرنامج. للحفاظ على استمرار عمل الموقع للزبائن:

### الطريقة السريعة:
1.  شغل أمر `ngrok http 3000` في مجلد `server-nestjs`.
2.  انسخ الرابط الجديد `https` (مثال: `https://tsunamic-unshameable-maricruz.ngrok-free.dev`).
3.  اذهب إلى مشروعك على **[Vercel Dashboard](https://vercel.com/dashboard)**.
4.  انتقل إلى **Settings** > **Environment Variables**.
5.  ابحث عن المتغير `VITE_API_URL`.
6.  اضغط **Edit** واستبدل القيمة القديمة بـ: `https://your-new-ngrok-url.ngrok-free.dev/api`
7.  كرر نفس الخطوات لـ `VITE_API_BASE_URL`.
8.  اضغط **Save**. (التغيير يسري فوراً، لا حاجة لعمل Redeploy).

### 📖 للمزيد من التفاصيل:
راجع ملف **[VERCEL_SETUP.md](./VERCEL_SETUP.md)** للحصول على دليل شامل.

## 🧠 المنطق البرمجي (System Logic)

1.  **المستخدم** يفتح الموقع (على فيرسال أو محلياً).
2.  **الموقع** يرسل طلبات البيانات (API Requests) إلى رابط `VITE_API_URL` (الذي هو رابط ngrok).
3.  **Ngrok** يستقبل الطلب ويمره فوراً إلى سيرفرك المحلي (`localhost:3000`).
4.  **NestJS (السيرفر)** يعالج الطلب:
    *   إذا كان طلب بيانات: يجلبها من **Supabase**.
    *   إذا كان طلب واتساب: يتحدث مع مكتبة **Baileys** لإرسال/استقبال الرسائل.
5.  **الرد:** يعود المسار بالعكس حتى يظهر على شاشة المستخدم.
